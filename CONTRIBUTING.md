# Contributing

## Scope
Contributions are welcome for:
- Content edits and additions to chapters
- Figures and educational assets
- Build/publishing improvements
- Issue triage and documentation

## Workflow
1. Open an issue describing the change.
2. Create a feature branch.
3. Submit a pull request (PR) to `main`.
4. Ensure CI passes.

## Commit conventions
Prefer Conventional Commits:
- `feat:` new content/features
- `fix:` corrections
- `docs:` documentation-only changes
- `chore:` maintenance (build, CI, repo)

## Local preview
- Install Quarto.
- Run: `quarto render manuscript`

## Style
- Keep chapters modular.
- Use clear headings and short paragraphs.
- Prefer figure sources that can be reproduced.
